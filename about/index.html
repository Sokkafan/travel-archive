About

关于

这个站点用于长期整理旅行地点、访问记录、文章和待补事项。

第一版是文件驱动的静态站：新增地点需要新建 Markdown 文件，提交后重新部署。

地点模型已经支持多次访问。地图保持一个地点一个标记，地点详情页展示每次访问的时间线、路线和备注。

新增地点流程

  1. 在 src/content/travels/ 新建一个 Markdown 文件。
  2. 填写地点层级、坐标、标签、摘要和 visits 访问记录。
  3. 正文补充印象、路线、照片和实用信息。
  4. 提交并构建，首页、足迹树、地图和标签会自动更新。
